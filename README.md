# Счёт отжиманий — приложение для Android

Счётчик-тренер по отжиманиям: программа на 52 недели, режимы «Свободно» и «На время»,
хваты с диаграммами, голосовой счёт, метроном, история, рекорды и **напоминания о тренировке**.

Это проект-обёртка [Capacitor](https://capacitorjs.com): само приложение — в папке `www/`
(один файл `index.html`), а Android-проект — в `android/`.

---

## Способ 1 (рекомендую) — собрать APK через GitHub без установки чего-либо

Ничего ставить на компьютер не нужно. Всё собирается в облаке GitHub.

1. Создай аккаунт на https://github.com (если нет) и нажми **New repository** → назови, например, `otzhimaniya`, сделай **Public** или **Private** → **Create**.
2. Загрузи в репозиторий **всё содержимое этой папки**:
   - Проще всего: на странице репозитория → **Add file → Upload files** → перетащи сюда все файлы и папки (`www`, `android`, `.github`, `package.json`, `capacitor.config.json`, и т.д.) → **Commit changes**.
   - Или через git:
     ```bash
     git init
     git add .
     git commit -m "init"
     git branch -M main
     git remote add origin https://github.com/ТВОЙ_ЛОГИН/otzhimaniya.git
     git push -u origin main
     ```
3. Открой вкладку **Actions** в репозитории. Сборка **Build APK** запустится сама после загрузки.
   Если не запустилась — нажми **Build APK → Run workflow**.
4. Подожди ~3–6 минут. Когда галочка позеленеет, открой завершившийся запуск → внизу блок **Artifacts** → скачай **`otzhimaniya-debug-apk`** (это zip с файлом `app-debug.apk`).
5. Распакуй, перекинь `app-debug.apk` на телефон и установи (нужно разрешить «Установка из неизвестных источников»).

При первом запуске приложение спросит разрешение на уведомления — разреши, чтобы напоминания приходили.

## Способ 2 — собрать локально (нужен Android Studio / Android SDK)

Требуется: Node.js 20+, JDK 21, Android SDK (через Android Studio).

```bash
npm install
npx cap sync android
cd android
./gradlew assembleDebug        # на Windows: gradlew.bat assembleDebug
```

Готовый файл: `android/app/build/outputs/apk/debug/app-debug.apk`.

Открыть проект в Android Studio (удобно для запуска на эмуляторе/телефоне):
```bash
npx cap open android
```

## Способ 3 — без APK, просто «как приложение» (PWA)

`www/` — это полноценный PWA. Если выложить папку на любой https-хостинг
(GitHub Pages, Netlify, Vercel) и открыть на телефоне в Chrome →
меню → **Добавить на главный экран**. Запускается как приложение, работает офлайн.
Уведомления в этом режиме менее надёжны, чем в APK (зависит от браузера и системы).

---

## Как обновлять приложение

Поменял что-то в `www/index.html` → закоммить/залей в GitHub → новая сборка APK создастся
автоматически (Способ 1). Локально: `npx cap sync android` и пересобрать (Способ 2).

## Где что лежит

```
www/                     приложение (index.html, манифест, sw.js, иконки)
android/                 нативный Android-проект (Capacitor)
capacitor.config.json    appId, имя приложения, настройки плагинов
package.json             зависимости (Capacitor + плагин уведомлений)
.github/workflows/       авто-сборка APK на GitHub
```

## Про уведомления

Напоминания используют плагин `@capacitor/local-notifications` — это **нативные** уведомления,
они приходят даже когда приложение полностью закрыто. Включается в приложении:
**⚙ Настройки → Тренировка → Напоминание** + время и дни тренировок.
Приложение планирует напоминания на 14 дней вперёд и обновляет их при каждом открытии.

## Подпись релизной версии (по желанию)

Сборка выше — **debug**-версия (подписана отладочным ключом, ставится на телефон для себя).
Для публикации в Google Play нужен release-APK/AAB со своим ключом — это отдельный шаг,
напиши, если понадобится, добавлю.
